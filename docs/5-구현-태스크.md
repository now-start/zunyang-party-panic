## 1. 지금 고정한 전제

- 패키지 구조는 `adapter / application / domain / config`
- `application`, `domain`은 순수 Java
- `libGDX`는 `adapter`, `config`에서 적극 사용
- 실이미지가 없어도 구현을 진행할 수 있도록 `semantic image key -> 실제 파일 or placeholder` 구조로 간다
- 이후 실제 이미지 교체는 `파일 추가` 또는 `경로 매핑 수정`만으로 끝나야 한다

## 5. placeholder 이미지 전략

- 화면은 절대 직접 파일 경로를 참조하지 않는다
- 화면은 `SampleTextureId` 같은 의미 키만 사용한다
- 의미 키는 `실제 이미지 경로`에 매핑된다
- 파일이 없으면 `Pixmap`으로 생성한 placeholder texture를 대신 쓴다
- 실제 이미지가 들어오면 같은 키의 파일만 채워 넣으면 된다
