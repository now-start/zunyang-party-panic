# Game Screen Wireframe

- Version: 0.1
- Date: 2026-04-16
- Based on: `docs/03-birthday-game-loop-design.md`, `docs/04-streamer-intervention-design.md`, `docs/05-content-table-and-ui-action-mapping.md`
- Goal of this document: Windows 클라이언트 기준으로 방송 노출 영역과 스트리머 조작 영역을 구분한 와이어프레임을 정의한다.

## 1. 이번 단계에서 정할 것

이번 문서에서는 아래를 정리한다.

- 전체 창 구조
- 방송에 잡히는 메인 게임 화면
- 스트리머만 보는 조작 패널
- 각 페이즈별 화면 레이아웃
- 저채팅량 환경에서 화면이 비어 보이지 않게 하는 장치

## 2. 화면 설계 전제

현재 전제는 아래와 같다.

- 플랫폼: Windows
- 방송 운영 대상
- 채팅량: 분당 약 10개
- 고유 시청자 수: 약 40명 내외
- 스트리머 개입 버튼이 필요함

이 전제 때문에 화면은 반드시 아래 조건을 만족해야 한다.

- 방송 화면만 봐도 현재 해야 할 행동이 보일 것
- 스트리머는 별도 조작 영역에서 즉시 버튼을 누를 수 있을 것
- 채팅이 적어도 화면이 비어 보이지 않을 것
- OBS에서 자르기 쉬운 구조일 것

## 3. 추천 창 구조

권장 구조는 `단일 Windows 클라이언트 + 방송 영역/조작 영역 분리`다.

즉, 한 프로그램 안에 두 영역이 있다.

- 좌측: 방송 노출용 메인 게임 영역
- 우측: 스트리머 전용 조작 패널

권장 비율은 아래와 같다.

- 방송 영역: 75% ~ 80%
- 조작 패널: 20% ~ 25%

OBS에서는 좌측 방송 영역만 캡처하거나, 전체 창을 잡고 우측 패널을 크롭하는 구조가 적절하다.

## 4. 권장 해상도 기준

### 기본 작업 해상도

- 전체 클라이언트: `1920 x 1080`

### 방송 캡처 기준

- 방송 메인 영역: 약 `1500 x 1080` 또는 `1520 x 1080`
- 스트리머 패널: 약 `400 x 1080`

### 최소 대응 해상도

- 전체 클라이언트 최소: `1600 x 900`

## 5. 전체 레이아웃 와이어프레임

아래는 권장 기본 구조다.

```text
+--------------------------------------------------------------------------------------------------+
|                                      TOP STATUS BAR                                              |
|  ROUND / TIMER / PARTY NAME / PREP / STABILITY / CHEER                                           |
+-----------------------------------------------------------------------------------+--------------+
|                                                                                   |              |
|                                                                                   |   STREAMER   |
|                                                                                   |   ACTION     |
|                                MAIN PARTY STAGE                                   |   PANEL      |
|                                                                                   |              |
|                                                                                   |  - Start     |
|                                                                                   |  - Pick      |
|                                                                                   |  - Reroll    |
|                                                                                   |  - Skill     |
|                                                                                   |  - Finale    |
|                                                                                   |              |
|                                                                                   |  STATUS      |
|                                                                                   |  LOG         |
+-----------------------------------------------------------------------------------+--------------+
|                                COMMAND / CHAT PROMPT BAR                                        |
|                    NOW TYPE: 1 / 2 / 3    or    !축하 / !응원                                   |
+--------------------------------------------------------------------------------------------------+
```

핵심은 아래 두 가지다.

- 방송 화면은 가운데 큰 무대가 중심
- 조작 패널은 오른쪽에 붙지만 방송 캡처에서는 제외 가능

## 6. 방송 노출 영역 고정 요소

방송 영역에는 페이즈가 바뀌어도 항상 보여야 하는 정보가 있다.

### 상단 고정 HUD

- 현재 라운드 이름
- 남은 시간
- 준비도
- 안정도
- 축하열기
- 현재 파티 이름 또는 현재 판 제목

### 하단 고정 커맨드 바

- 현재 입력 가능한 명령어
- 짧은 설명
- 동률 또는 무응답 처리 안내가 뜰 수 있는 영역

### 중앙 메인 스테이지

- 케이크
- 파티 배경
- 사고 오브젝트
- 피날레 연출
- 결과 화면 비주얼

즉, 중앙은 항상 `지금 판의 감정 중심`을 담당해야 한다.

## 7. 스트리머 조작 패널 고정 요소

우측 패널은 매 페이즈마다 기능은 바뀌되, 구조는 최대한 고정해야 한다.

### 상단

- 현재 라운드
- 다음 라운드 예고
- 남은 개입 횟수

### 중단

- 현재 사용할 수 있는 버튼만 강조 표시
- 비활성 버튼은 흐리게 처리

### 하단

- 시스템 로그
- 최근 채팅 유효 입력 수
- fallback 작동 여부

권장 구조는 아래와 같다.

```text
+----------------------+
| ROUND STATUS         |
| - Phase 2            |
| - 12s left           |
| - Skill 1/1          |
+----------------------+
| ACTIONS              |
| [Start]              |
| [Today's Pick]       |
| [Reroll]             |
| [Birthday Wish]      |
| [Finale Trigger]     |
+----------------------+
| LIVE INFO            |
| voters: 7            |
| tie: no              |
| fallback: off        |
+----------------------+
| COMMENT PRESET       |
| [Best Moment]        |
| [Chaos Pick]         |
+----------------------+
```

## 8. Phase 0 와이어프레임

오프닝은 간단해야 한다.  
시작 직전의 긴장감만 주고 바로 넘어가면 된다.

```text
+-----------------------------------------------------------------------------------+--------------+
| ROUND 0: START                                                    TIMER 05        |  Start       |
+-----------------------------------------------------------------------------------+--------------+
|                                                                                   |              |
|                                 BIRTHDAY PARTY GAME                               |  READY       |
|                                                                                   |  - open      |
|                                 5                                                 |  - audio ok  |
|                                 4                                                 |  - input ok  |
|                                 3                                                 |              |
|                                                                                   |              |
+-----------------------------------------------------------------------------------+--------------+
|                         곧 생일 파티 준비 시작 / 채팅 대기 중                                         |
+--------------------------------------------------------------------------------------------------+
```

핵심:

- 중앙은 큰 숫자 카운트다운
- 패널은 시작 상태 확인

## 9. Phase 1~2 선택 라운드 와이어프레임

선택 라운드는 `보기 3개`, `실시간 득표`, `오늘의 픽`, `명령어 안내`가 모두 잘 보여야 한다.

```text
+-----------------------------------------------------------------------------------+--------------+
| ROUND 1: CAKE BASE                                              TIMER 18          | Pick: 2      |
| PREP 32   STABILITY 50   CHEER 10                                                  | Reroll: ON   |
+-----------------------------------------------------------------------------------+--------------+
|                                                                                   |              |
|                            [ CURRENT PARTY VISUAL PREVIEW ]                       | ACTIONS      |
|                                                                                   | [Pick #2]    |
|                                                                                   | [Reroll]     |
|    +------------------+   +------------------+   +------------------+             |              |
|    | 1. 딸기          |   | 2. 초코          |   | 3. 치즈          |             | voters: 6    |
|    | vote 35%         |   | vote 45%         |   | vote 20%         |             | tie: no      |
|    |                  |   | TODAY'S PICK     |   |                  |             |              |
|    +------------------+   +------------------+   +------------------+             |              |
|                                                                                   |              |
+-----------------------------------------------------------------------------------+--------------+
|                           채팅 입력: 1 / 2 / 3   |   동률 시 스트리머 판정                              |
+--------------------------------------------------------------------------------------------------+
```

선택 라운드 핵심:

- 3개 박스는 충분히 크게
- `TODAY'S PICK`은 즉시 눈에 들어와야 함
- 현재 누적된 파티 비주얼이 뒤쪽에서 계속 보이면 좋음

## 10. Phase 3 사고 대응 와이어프레임

사고 대응은 저채팅량을 고려해 `3회 사건`, `각 사건 5~6초` 구조로 간다.  
따라서 초고속 액션 UI보다 `현재 사건이 무엇인지`가 또렷해야 한다.

```text
+-----------------------------------------------------------------------------------+--------------+
| ROUND 3: TROUBLE EVENT                                         TIMER 06           | Skill: READY |
| PREP 55   STABILITY 42   CHEER 12                                                  | Use Wish     |
+-----------------------------------------------------------------------------------+--------------+
|                                                                                   |              |
|                                 [ BIG TROUBLE BANNER ]                            | ACTIONS      |
|                                  고양이가 케이크로 점프!                           | [Wish]       |
|                                                                                   | [Protect]    |
|                 1. LEFT                  2. CENTER                3. RIGHT        |              |
|                                                                                   | voters: 4    |
|                   [ danger ]               [ CAT ]                [ safe ]         | fallback:no  |
|                                                                                   |              |
|                              current vote -> CENTER 60%                           |              |
|                                                                                   |              |
+-----------------------------------------------------------------------------------+--------------+
|                        채팅 입력: 1 / 2 / 3   |   지금 막으면 안정도 유지                               |
+--------------------------------------------------------------------------------------------------+
```

사고 대응 핵심:

- 중앙 사건 설명이 커야 함
- 정답 찾기보다 "지금 무슨 일이 벌어졌는지"가 먼저 읽혀야 함
- 스트리머 스킬 버튼은 이때 가장 눈에 띄게

## 11. Phase 4 피날레 와이어프레임

피날레는 채팅량이 적더라도 화면상 폭발감이 살아야 한다.  
그래서 게이지가 충분히 크고, 텍스트가 단순해야 한다.

```text
+-----------------------------------------------------------------------------------+--------------+
| ROUND 4: FINALE                                               TIMER 20            | Finale LOCK  |
| PREP 68   STABILITY 49   CHEER 61                                                  |              |
+-----------------------------------------------------------------------------------+--------------+
|                                                                                   | ACTIONS      |
|                                  생일 축하 피날레                                  | [Finale]     |
|                                                                                   |              |
|                         +-------------------------------------+                     | unique: 7    |
|                         |          CHEER GAUGE 61%            |                     | total: 11    |
|                         +-------------------------------------+                     | ready: no    |
|                                                                                   |              |
|                               !축하      !응원                                    |              |
|                                                                                   |              |
|                              배경 반짝임 / 풍선 / 폭죽 예고                         |              |
|                                                                                   |              |
+-----------------------------------------------------------------------------------+--------------+
|                      채팅 입력: !축하 / !응원   |   참여 인원 수가 가장 중요                              |
+--------------------------------------------------------------------------------------------------+
```

피날레 핵심:

- 중앙 게이지는 압도적으로 커야 함
- 채팅이 무엇을 입력해야 하는지 큰 텍스트로 반복 표시
- 스트리머는 게이지 임계치 도달 시 `Finale` 버튼을 직접 누름

## 12. Phase 5 결과 화면 와이어프레임

결과 화면은 숫자 나열이 아니라 `결과 한 장`처럼 보여야 한다.

```text
+-----------------------------------------------------------------------------------+--------------+
| RESULT SCREEN                                                                     | Comment      |
+-----------------------------------------------------------------------------------+--------------+
|                                                                                   | [Best Moment]|
|                                FINAL GRADE : A                                    | [Chaos Pick] |
|                                                                                   |              |
|                            [ COMPLETED PARTY HERO VISUAL ]                        |              |
|                                                                                   | viewers: 11  |
|                         오늘은 조금 시끄러웠지만 아주 좋은 파티였어요               | chats: 19    |
|                                                                                   |              |
|          케이크: 초코     장식: 폭죽 토퍼     사고 대응: 2 / 3 성공                |              |
|                                                                                   |              |
+-----------------------------------------------------------------------------------+--------------+
|                     다시 하기 / 다음 판 준비 / 스트리머 코멘트 표시 영역                             |
+--------------------------------------------------------------------------------------------------+
```

결과 화면 핵심:

- 등급과 한 줄 멘트를 크게
- 선택 결과 요약은 보조 정보로
- 스트리머 코멘트가 결과 카드 일부처럼 붙어야 함

## 13. 저채팅량 대응 시각 장치

현재 프로젝트에서 아주 중요하다.

채팅이 적으면 UI가 빈 느낌이 나기 쉬우므로 아래 장치가 필요하다.

### 1. 투표 수보다 상태 변화를 크게 보여 주기

- 퍼센트 바
- 오브젝트 변화
- 게이지 변화

### 2. 최근 반응 수를 부드럽게 표시

- `현재 6명 참여 중`
- `이번 사건 4표 집계`

이런 식으로 절대 수치가 작아도 의미 있어 보이게 한다.

### 3. 무응답 배너를 민망하지 않게 처리

예시:

- `스트리머 판정 진행`
- `소수 정예 투표 반영`
- `간신히 막아냈어요`

직접적으로 “사람이 적다”는 인상은 피하는 편이 좋다.

## 14. 방송 캡처 안전 구역

OBS에서 잘라 쓰는 것을 고려하면 아래 구역을 안전 구역으로 본다.

### 방송 노출 안전 구역

- 좌측 75% 영역
- 상단 HUD
- 중앙 메인 스테이지
- 하단 명령어 바

### 방송 비노출 권장 구역

- 우측 스트리머 패널 전체

즉, 우측 패널에 중요한 방송 정보가 들어가면 안 된다.  
방송에 보여야 하는 핵심 정보는 모두 좌측에 있어야 한다.

## 15. 권장 첫 구현안

초기 MVP 와이어프레임 기준으로는 아래가 가장 안전하다.

- 전체 1920x1080 창 1개
- 좌측 메인 방송 영역
- 우측 스트리머 액션 패널
- 상단 고정 HUD
- 하단 고정 명령어 바
- 중앙은 각 페이즈별 큰 스테이지 화면

이 구조는 구현도 과하지 않고, 방송 운용도 쉽다.

## 16. 이번 단계 결론

현재 기준 화면 구조는 아래 문장으로 정리할 수 있다.

`이 게임의 Windows 클라이언트는 좌측 방송 노출용 메인 스테이지와 우측 스트리머 조작 패널로 나뉘며, 방송에 필요한 핵심 정보는 모두 좌측 안전 구역 안에서 해결한다.`
