# Birthday Chat Game Plan

- Version: 0.2
- Date: 2026-04-16
- Project id: `zunyang-party-panic`
- Target channel: 치즈냥 ZuNyang
- Project type: 생일기념 팬게임
- Client target: Windows
- Fixed now: 생일 콘셉트, 팬게임 톤, 시청자 채팅 상호작용
- Decide later: 장르, 개발 언어, 엔진, 최종 구현 방식

## 1. 이번 단계에서 확정할 것

지금은 게임 규칙을 완전히 고정하는 단계가 아니라, 이 게임이 어떤 감정과 어떤 방송 장면을 만들 게임인지 정하는 단계다.

이번 기획 단계의 목적은 아래 네 가지다.

- 이 게임이 `생일기념 팬게임`으로 어떤 의미를 가져야 하는지 정의
- 스트리머와 시청자가 어떤 관계로 참여하는지 정리
- 어떤 분위기와 어떤 장면이 핵심인지 결정
- 나중에 장르를 정해도 흔들리지 않을 `코어 컨셉`을 잡기

## 2. 프로젝트 전제

현재까지 확정된 전제는 아래와 같다.

- 목적은 `스트리머 생일을 기념하는 팬게임`
- 시청자 채팅 상호작용이 핵심
- 장르는 나중에 결정
- 개발 언어와 엔진도 나중에 결정
- 게임 클라이언트는 우선 `Windows`

즉, 지금은 "무슨 엔진으로 만들지"보다 "이 게임이 어떤 생일 경험을 만들어야 하는지"가 먼저다.

## 3. 채널 공개 정보 기준 해석

2026-04-16 기준 공개 채널 페이지에서 확인한 내용은 아래와 같다.

- 채널명은 `치즈냥 ZuNyang`
- 소개 문구는 `talk + 게임!! 매일 저녁 7시에 만나요`
- 최근 노출 태그는 `talk`, `버츄얼`, `수다`, `vtuber`, `종합게임`
- 최근 방송 제목과 흐름에서 `참여`, `!투표` 같은 시청자 개입형 패턴이 보인다
- 전체적으로 강한 경쟁보다 `리액션`, `수다`, `참여 분위기`가 중심인 채널로 읽힌다

이걸 기반으로 보면, 생일 팬게임의 방향은 아래 쪽이 맞다.

- 팬들이 같이 만들어 주는 생일 이벤트
- 스트리머가 리액션 하기 좋은 상황
- 룰보다 분위기와 참여감이 중요한 구조
- 승패보다 `축하하는 과정`이 핵심인 구조

## 4. 컨셉 키워드

이 프로젝트의 핵심 키워드는 아래처럼 잡는 것이 좋다.

- 생일
- 축하
- 팬 참여
- 소란스럽고 귀여운 파티
- 같이 만들어 가는 이벤트
- 방송 반응이 잘 나오는 작은 사고와 반전

반대로 피해야 하는 방향은 아래다.

- 지나치게 하드코어한 피지컬 중심
- 설명이 길고 복잡한 전략 중심
- 생일보다 경쟁이 먼저 보이는 구조
- 팬게임인데 스트리머와 채팅의 존재감이 약한 구조

## 5. 컨셉 후보 3안

장르를 지금 확정하지 않기 위해, 먼저 `무드와 판타지` 기준으로 3개 후보를 잡는다.

### A안. 생일 파티 준비 대작전

팬들이 힘을 합쳐 스트리머를 위한 생일 파티를 완성하는 콘셉트다.

- 핵심 판타지: `우리가 직접 생일상을 차려 준다`
- 핵심 감정: 기대감, 웃김, 마지막 축하의 폭발감
- 채팅 역할: 재료 선택, 장식 결정, 사고 수습, 마지막 축하 밀어주기
- 장점: 생일 테마가 가장 직관적이고, 채팅 참여 구조와 잘 붙는다
- 리스크: 너무 무난하게 가면 개성이 약해질 수 있다

### B안. 생일 케이크 대소동

완성 직전의 생일 케이크를 각종 사고에서 지켜 내는 콘셉트다.

- 핵심 판타지: `망해 가는 생일 파티를 구한다`
- 핵심 감정: 긴장감, 실수, 폭소, 기적 같은 성공
- 채팅 역할: 위기 대응, 빠른 선택, 사고 방지
- 장점: 방송적으로 반응이 세게 나온다
- 리스크: 방어 게임처럼 보이면 `생일기념 팬게임`보다 `미니게임` 느낌이 더 강해질 수 있다

### C안. 깜짝 생일 선물 전달 작전

팬들의 마음을 모아 스트리머에게 생일 선물을 전달한다는 콘셉트다.

- 핵심 판타지: `마음을 모아 특별한 선물을 완성한다`
- 핵심 감정: 귀여움, 몰입, 마지막 감동
- 채팅 역할: 선택, 응원, 선물 조합, 소원 모으기
- 장점: 팬게임이라는 의미가 가장 강하다
- 리스크: 잘못 잡으면 게임성보다 이벤트 연출에만 치우칠 수 있다

## 6. 추천 방향

현 단계에서 가장 적합한 메인 컨셉은 `A안. 생일 파티 준비 대작전`이다.

이유는 아래와 같다.

- `생일기념 팬게임`이라는 목적이 가장 선명하다
- 채팅 상호작용을 자연스럽게 여러 방식으로 넣을 수 있다
- 추후 어떤 장르로 가더라도 중심 설정이 흔들리지 않는다
- Windows 클라이언트 단일 실행 구조와도 잘 맞는다
- 방송에서 한 판씩 짧게 돌리기 좋은 장면 구성이 나온다

즉, 지금 단계에서는 `전투`나 `스테이지 클리어`보다 `팬들이 함께 생일 파티를 완성하는 경험`을 중심 축으로 잡는 것이 맞다.

## 7. 최종 컨셉 정의

### 컨셉 한 줄

`시청자 채팅이 힘을 모아 스트리머의 생일 파티를 완성하는 방송형 팬게임`

### 컨셉 설명

이 게임은 팬들이 채팅으로 생일 파티를 만들어 가는 축제형 팬게임이다.  
중요한 것은 높은 난이도나 깊은 시스템이 아니라, 채팅이 직접 결과를 만들고 스트리머가 그 결과에 리액션하는 과정 자체다.

### 이 게임이 만들어야 하는 장면

- 채팅이 한쪽 선택지로 몰리면서 분위기가 달아오르는 장면
- 예상 밖의 결과가 나와서 스트리머가 웃는 장면
- 마지막에 다 같이 축하를 밀어 넣는 장면
- 성공해도 좋고 망해도 웃긴 장면

### 이 게임의 정체성

- 축하형 팬게임
- 스트리머 반응형 게임
- 채팅 참여형 게임
- 생일 이벤트성 게임

## 8. 장르를 아직 정하지 않아도 되는 이유

지금 잡은 컨셉은 장르와 분리해서 유지할 수 있다.

예를 들어 나중에 아래 어느 방향으로 가도 컨셉은 유지된다.

- 파티 게임
- 캐주얼 실시간 액션
- 선택형 인터랙티브 게임
- 라운드 기반 협동 게임
- 간단한 시뮬레이션 게임

중요한 것은 장르가 아니라 아래 세 가지다.

- 팬들이 `생일을 함께 만들어 준다`는 감각
- 채팅이 실제로 `개입하고 있다`는 체감
- 스트리머가 `리액션 하기 좋은 결과`가 계속 나온다는 점

## 9. 컨셉 기준 필수 요소

장르가 바뀌어도 아래 요소는 유지하는 쪽이 좋다.

### 1. 팬들이 무언가를 완성한다

케이크든, 파티룸이든, 선물이든, 축하 게이지든 팬들이 함께 쌓아 가는 대상이 있어야 한다.

### 2. 채팅이 눈에 보이는 변화를 만든다

채팅 입력이 단순 숫자 집계로 끝나지 않고, 화면에 즉시 반영되어야 한다.

### 3. 실패도 재미있어야 한다

생일 팬게임이므로 실패가 짜증나면 안 된다.  
망해도 귀엽고, 웃기고, 다시 해 보고 싶어야 한다.

### 4. 마지막은 반드시 축하 장면이어야 한다

어떤 장르로 가더라도 마지막에는 `생일 축하`가 화면과 감정의 정점이 되어야 한다.

## 10. 아트와 분위기 방향

현 시점의 비주얼 톤은 아래 정도가 적절하다.

- 귀엽고 밝은 파티 무드
- 케이크, 촛불, 풍선, 선물상자 같은 직관적 생일 오브젝트
- 너무 화려한 경쟁 UI보다 방송 오버레이처럼 바로 읽히는 화면
- 보기만 해도 "생일 방송용 게임"이라고 느껴지는 명확한 테마

추천 무드는 아래와 같다.

- 사랑스럽다
- 정신없지만 즐겁다
- 팬들이 장난치면서도 결국 축하해 준다

## 11. Windows 클라이언트 기준 메모

현재 확정된 플랫폼이 Windows라는 점은 기획상 아래 의미를 가진다.

- 방송 PC에서 바로 실행 가능한 단일 클라이언트 구성이 유리하다
- 콘솔/모바일 제약을 먼저 고려할 필요가 없다
- 키보드, 마우스, 방송용 창 관리에 맞춘 연출을 생각하면 된다

하지만 이 단계에서는 여전히 아래는 보류한다.

- 엔진 선택
- 언어 선택
- 렌더링 방식
- 네트워크 구조
