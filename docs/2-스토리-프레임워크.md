# 스토리 프레임워크

- Version: 0.2
- Date: 2026-04-16
- Project id: `zunyang-party-panic`
- Purpose: 현재 프로젝트의 스토리 큰틀, 장르 정체성, 플레이타임 구조를 한 문서로 정리한다.

## 1. 한 줄 소개

`생일 방송을 앞둔 치즈냥이 자신의 준비방과 과거 생일의 기억 조각을 직접 정리하며, 오늘 어떤 분위기로 방송을 열지 스스로 정리해 가는 쯔꾸르풍 팬게임`

## 2. 핵심 전제

- 플레이어는 치즈냥을 직접 조작한다
- 치즈냥은 `생일을 맞는 사람`이면서 동시에 `오늘 생일 방송을 직접 여는 스트리머`다
- 게임의 중심은 채팅 시스템이 아니라 `탐색`, `조사`, `기억 조각`, `짧은 미니게임`, `방송 시작 직전의 정리`다
- 실제 인물의 속마음을 작가가 단정하지 않는다
- 인게임에서는 연도를 직접 명시하지 않고 `예전 생일의 기억`, `남아 있는 기록` 정도로 처리한다

## 3. 전통 쯔꾸르 문법

이번 프로젝트는 아래 기준으로 통일한다.

- 이동은 `4방향`만 사용한다
- 대각선 이동은 넣지 않는다
- 허브와 챕터 공간은 `타일 기반 맵`으로 구성한다
- 상호작용은 `플레이어 정면 한 칸 조사`를 기본으로 한다
- 대사는 `하단 대사창 + 초상화` 형식으로 처리한다
- 이벤트는 문, 책상, 우편함, 포토존 같은 `맵 이벤트` 중심으로 진행한다
- 미니게임은 허브에서 바로 뜨는 카드가 아니라 `맵 조사 후 진입하는 서브 장면`으로 연결한다

## 4. 이야기의 중심 질문

이 게임의 질문은 단순히 `준비를 끝낼 수 있는가`가 아니다.

`치즈냥은 오늘 생일 방송을 어떤 톤으로 시작하게 될까?`

그래서 전체 구조는 아래 두 축으로 돌아간다.

- 현재의 축: 오늘 생일 방송을 준비하는 이야기
- 기억의 축: 예전 생일의 기록과 편지, 장면 조각을 다시 확인하는 이야기

## 5. 장르와 감정 톤

장르:

- 쯔꾸르풍 탐색 게임
- 생일 팬게임
- 방송 준비 게임
- 아카이브 기반 감성 게임
- 짧은 미니게임이 들어간 스토리 게임

감정 톤:

- 시작: 조용함, 망설임
- 중반: 정리, 실감, 화면이 살아나는 느낌
- 후반: 기억 회수, 여운
- 피날레: 방송을 열 수 있겠다는 정리

## 6. 공간 구조

### 생일 방송 준비방

- 방송 책상
- 케이크 테이블
- 포토존
- 팬레터 우편함
- 잠긴 문들

역할:

- 첫 허브
- 오늘 방송 직전의 미완성 상태 제시
- 초반 탐색과 기본 준비

### 장식 창고

- 리본
- 전구
- 인형
- 초
- 상자

역할:

- 중반 확장 공간
- 케이크, 포토존, 무대에 필요한 재료 확보
- 치즈냥의 취향과 감각을 보여 주는 장소

### 백스테이지 복도

- 오래된 소품
- 짧은 메모
- 액자
- 프로젝터
- 기억 조각

역할:

- 기억 회수 파트
- 준비에서 회상으로 넘어가는 브리지
- 팬레터와 피날레 사이 감정 연결

### 생일 방송 무대

역할:

- 마지막 시작 지점
- 지금까지 정리한 것들이 한 화면에서 모이는 피날레

## 7. 플레이타임 목표

- 메인 루트: `95~105분`
- 천천히 조사하고 대사를 다 읽는 플레이: `105~120분`

이 시간을 억지 전투나 긴 미니게임으로 늘리지 않는다.
`탐색 + 조사 + 짧은 이벤트 + 미니게임 + 기억 회수`의 합으로 만든다.

## 8. 전체 챕터 구조

| 순서 | 챕터 | 예상 시간 | 역할 |
|---|---|---:|---|
| 0 | 프롤로그: 불이 덜 켜진 준비방 | 10분 | 첫 인상, 방송 시작 목표 제시 |
| 1 | 방송 책상의 불을 켜다 | 12분 | 방송 리듬 회복 |
| 2 | 장식 창고를 열다 | 14분 | 공간 확장, 소품 확보 |
| 3 | 케이크가 가운데에 놓이는 순간 | 12분 | 오늘 방송의 중심 장면 완성 |
| 4 | 포토존에 오늘을 남기다 | 12분 | 화면에 남길 장면 정리 |
| 5 | 조명이 닿지 않는 복도 | 14분 | 기억 회수 브리지 |
| 6 | 팬레터를 열다 | 18분 | 남아 있던 기록 정리 |
| 7 | 생일 방송 무대 | 8분 | 피날레와 엔딩 |

## 9. 기억 조각 사용 원칙

- 개발 참고 자료는 과거 생일 방송 정보, 팬레터, 팬영상이다
- 하지만 게임 안에서는 `몇 년도`라고 직접 부르지 않는다
- `첫 기억`, `쌓여 온 기억`, `가장 가까운 기억` 정도의 감정 층위로 다룬다
- 완전한 연대기가 아니라, 지금 손에 남아 있는 일부 기록만 열리는 구조가 맞다

## 10. 엔딩 방향

엔딩은 성공/실패보다 `얼마나 깊게 정리했는가`의 차이로 나뉜다.

- 조용한 엔딩
- 따뜻한 엔딩
- 진심 엔딩

공통점은 같다.

`이제는 오늘 방송을 시작할 수 있겠다`

## 11. 구현 기준

- 허브 조사 텍스트는 전부 치즈냥 시점 독백
- 방송 책상은 `채팅/투표`가 아니라 `방송 첫 화면과 리듬 정리`
- 케이크는 `사고 수습`보다 `오늘의 중심 오브젝트 완성`
- 포토존은 `점수 경쟁`보다 `방송 화면에 남길 컷 정리`
- 팬레터는 `감정 해석 강요`보다 `기록을 다시 확인하는 구조`
- 피날레는 `카메라 앞에 설 준비가 되었다`는 느낌으로 마무리
- 허브 화면은 큰 일러스트 스팟 방식이 아니라 `전통 2D 타일 허브`로 구현한다
- 대사 연출은 설명 카드가 아니라 `RPG Maker식 대사창`을 우선한다
