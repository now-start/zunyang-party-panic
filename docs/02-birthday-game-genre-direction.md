# Birthday Game Genre Direction

- Version: 0.1
- Date: 2026-04-16
- Based on: `docs/01-birthday-chat-game-plan.md`
- Goal of this document: 생일기념 팬게임에 맞는 장르 후보를 좁히고, 각 후보에 맞는 채팅 상호작용 방식을 비교한 뒤 우선 추천 방향을 정한다.

## 1. 이번 단계에서 결정할 것

이번 단계는 장르를 완전히 확정하는 문서라기보다, 실제로 방송에서 굴릴 수 있는 방향만 남기는 단계다.

이번 문서에서 정리할 내용은 아래와 같다.

- 장르 후보 3안
- 각 장르에 맞는 채팅 상호작용 방식
- 방송 적합성 기준 비교
- 현재 기준 추천 1안

## 2. 장르 선정 기준

이번 프로젝트의 장르는 아래 기준으로 판단하는 것이 맞다.

- 생일기념 팬게임이라는 목적에 잘 맞는가
- 시청자 채팅이 결과를 바꾸는 체감이 강한가
- 스트리머 리액션 포인트가 자주 나오는가
- 설명이 짧고 방송 중 반복 플레이가 가능한가
- Windows 단일 클라이언트로 운영하기 무리가 없는가

즉, 깊은 시스템보다 `짧은 판`, `즉각 반응`, `생일 분위기`, `채팅 개입감`이 더 중요하다.

## 3. 장르 후보 3안

### A안. 라운드형 협동 파티 시뮬레이션

팬들이 채팅으로 파티 준비 요소를 하나씩 결정하며 생일 파티를 완성하는 장르다.

- 장르 인상: 파티 게임 + 가벼운 시뮬레이션
- 플레이 감각: 선택, 누적, 결과 확인
- 핵심 재미: 채팅이 다 같이 파티를 만든다는 감각
- 생일 적합도: 매우 높음

#### 채팅 상호작용 방식

- `1`, `2`, `3`으로 재료/장식/이벤트 선택
- `!응원`, `!축하`로 축제 게이지 증가
- 특정 라운드에서 다수결 결과가 화면에 즉시 반영

#### 방송 장점

- 룰 설명이 쉽다
- 스트리머가 계속 멘트를 얹기 좋다
- 성공과 실패 모두 화면 리액션이 잘 나온다
- 팬게임다운 의미가 가장 분명하다

#### 리스크

- 너무 선택지만 반복하면 단조로워질 수 있다
- 타격감이 약하면 게임보다 이벤트처럼 보일 수 있다

### B안. 실시간 협동 미니액션

채팅이 실시간으로 사고를 막거나 특정 위치를 방어하면서 파티를 지켜 내는 장르다.

- 장르 인상: 캐주얼 액션 + 방어형 미니게임
- 플레이 감각: 급한 판단, 짧은 긴장, 빠른 성공/실패
- 핵심 재미: 채팅이 즉시 반응해서 위기를 넘기는 감각
- 생일 적합도: 중상

#### 채팅 상호작용 방식

- `1`, `2`, `3`으로 좌/중/우 방어
- `!밀어`, `!막아` 같은 짧은 명령어로 집단 반응
- 1초 단위 또는 2초 단위 집계로 방어 성공 판정

#### 방송 장점

- 텐션이 빠르게 오른다
- 클립 포인트가 잘 나온다
- 채팅 반응 체감이 가장 강하다

#### 리스크

- 생일보다 액션이 먼저 보일 수 있다
- 채팅 속도와 집계 구조를 잘못 잡으면 혼란스럽다
- 반복 플레이 시 피로감이 빨리 올 수 있다

### C안. 선택형 생일 이벤트 어드벤처

채팅이 여러 상황에서 선택을 내려 생일 이벤트를 원하는 방향으로 이끄는 장르다.

- 장르 인상: 인터랙티브 스토리 + 이벤트 진행형
- 플레이 감각: 선택, 분기, 결과 감상
- 핵심 재미: 예상 밖의 전개와 엔딩 차이
- 생일 적합도: 높음

#### 채팅 상호작용 방식

- `1`, `2`, `3`으로 상황 선택
- 특정 장면에서 `!축하`, `!선물`, `!촛불` 같은 테마형 입력
- 누적 선택에 따라 엔딩 분기

#### 방송 장점

- 세계관과 캐릭터성을 녹이기 좋다
- 팬들이 “우리 선택으로 이런 결과가 나왔다”는 감각을 받기 쉽다
- 감동형 또는 개그형 엔딩을 설계하기 좋다

#### 리스크

- 템포가 느려지면 수다 방송 보조 콘텐츠처럼 느껴질 수 있다
- 시스템보다 연출 의존도가 높아진다
- 분기 설계가 늘어날수록 제작량이 커진다

## 4. 후보 비교

### 생일 팬게임 적합성

- A안: 가장 적합
- B안: 보조 장르로 적합
- C안: 테마 측면에서는 강하지만 게임 텐션은 약해질 수 있음

### 채팅 개입 체감

- A안: 높음
- B안: 매우 높음
- C안: 중상

### 방송 템포

- A안: 안정적
- B안: 빠름
- C안: 느긋함

### 리액션 포인트

- A안: 넓고 고르게 나옴
- B안: 강한 순간형
- C안: 결과 장면 중심

### 운영 안정성

- A안: 가장 안정적
- B안: 집계와 타이밍 설계가 중요
- C안: 콘텐츠 소모가 빠를 수 있음

## 5. 추천 방향

현재 기준 추천 장르는 `A안. 라운드형 협동 파티 시뮬레이션`이다.

다만 완전히 정적인 선택형으로 가기보다는, `B안의 짧은 실시간 대응 요소`를 일부 섞는 방향이 가장 좋다.

즉, 현재 추천 구조는 아래처럼 정리할 수 있다.

- 메인 장르: 라운드형 협동 파티 시뮬레이션
- 보조 장치: 짧은 실시간 위기 대응 미니액션
- 목표 경험: 팬들이 함께 생일 파티를 완성하고, 중간중간 작은 사고를 수습하며, 마지막에 다 같이 축하를 폭발시키는 경험

이 방향이 좋은 이유는 아래와 같다.

- 생일 팬게임이라는 목적이 가장 또렷하다
- 채팅이 선택과 반응 두 방식으로 모두 참여할 수 있다
- 한 판당 흐름이 명확해서 방송 진행이 쉽다
- Windows 클라이언트 단일 화면 구성에도 잘 맞는다
- 이후 장르 디테일을 바꿔도 전체 컨셉이 흔들리지 않는다

## 6. 추천 채팅 인터랙션 원칙

장르를 위 방향으로 가져간다면 채팅 상호작용은 아래 원칙으로 가는 편이 좋다.

### 원칙 1. 입력은 짧고 단순해야 한다

- 기본은 `1`, `2`, `3`
- 보조 입력은 `!축하`, `!응원` 정도만 사용

### 원칙 2. 결과는 즉시 보여야 한다

- 다수결 결과가 바로 오브젝트 변화로 보이기
- 게이지, 퍼센트, 효과음 등으로 즉시 피드백 주기

### 원칙 3. 같은 입력이라도 라운드 의미는 달라도 된다

예를 들어 `1`, `2`, `3`은 라운드마다 다른 선택지를 가리켜도 된다.  
중요한 것은 채팅이 외우기 쉬운 입력 체계를 유지하는 것이다.

### 원칙 4. 마지막에는 전체 채팅이 한 방향으로 몰리는 구간이 필요하다

생일 게임인 만큼 마지막에는 모두가 같은 입력을 치며 축하하는 피날레가 있어야 한다.

## 7. 이번 단계 결론

현재 프로젝트의 우선 장르 방향은 아래 문장으로 정리할 수 있다.

`이 게임의 우선 장르 방향은, 채팅으로 생일 파티를 함께 완성하는 라운드형 협동 파티 시뮬레이션이며, 방송 텐션을 위해 짧은 실시간 대응 미니액션을 일부 결합하는 형태다.`
