# Content Table And UI Action Mapping

- Version: 0.1
- Date: 2026-04-16
- Based on: `docs/03-birthday-game-loop-design.md`, `docs/04-streamer-intervention-design.md`
- Goal of this document: 저채팅량 기준으로 실제 선택지, 사고 이벤트, 결과 연출, 스트리머 액션 버튼을 콘텐츠 테이블 형태로 정리한다.

## 1. 운영 기준값

현재 운영 가정은 아래와 같다.

- 평시 채팅량: 분당 약 10개
- 고유 시청자 수: 약 40명 내외
- 실제 게임 참여 채팅 인원: 약 8명 ~ 15명 예상

이 수치 기준으로 보면 게임은 `대규모 채팅 난전`이 아니라 `소수 참여자도 판을 굴릴 수 있는 방송형 선택 게임`으로 설계해야 한다.

즉, 콘텐츠는 아래 조건을 만족해야 한다.

- 한 번 보고 바로 이해되는 3지선다
- 적은 표로도 결과가 나오는 구조
- 동률이나 무응답이 나와도 방송이 안 멈추는 구조
- 스트리머 리액션과 개입으로 빈 구간을 채울 수 있는 구조

## 2. 저채팅량 기준 콘텐츠 설계 원칙

### 원칙 1. 선택지는 설명 없이 읽혀야 한다

예를 들어 아래가 좋다.

- 딸기 / 초코 / 치즈
- 분홍 / 노랑 / 하늘
- 리본 / 초 / 폭죽

반대로 아래는 피하는 것이 좋다.

- 설명이 길어야 의미가 보이는 선택지
- 옵션 이름만 보고 결과가 상상되지 않는 선택지

### 원칙 2. 선택지마다 리액션 포인트가 달라야 한다

셋 다 무난하면 채팅이 굳이 반응할 이유가 줄어든다.

- 하나는 정석
- 하나는 귀여움
- 하나는 장난기 또는 사고 유발

이런 구조가 좋다.

### 원칙 3. 무응답 대비 규칙이 반드시 있어야 한다

예상보다 채팅이 적으면 아래 규칙으로 판을 굴린다.

- 유효 투표 3명 미만: 스트리머가 `오늘의 픽` 또는 수동 선택
- 동률: 스트리머가 최종 판정
- 사고 대응 구간 무응답: 기본 실패 대신 `반실패` 처리

## 3. 준비 선택 라운드 콘텐츠 테이블

준비 선택 라운드는 2개 라운드만 사용하므로, 각 라운드마다 보기 구성이 즉시 이해되는 것이 중요하다.

### Phase 1 후보 테이블

| 세트 | 선택 주제 | 1번 | 2번 | 3번 | 리액션 포인트 |
|---|---|---|---|---|---|
| A | 케이크 베이스 | 딸기 | 초코 | 치즈 | 정석 vs 무난 vs 취향차 |
| B | 메인 컬러 | 분홍 | 노랑 | 하늘 | 귀여움과 화면 톤 차이 |
| C | 파티 테마 | 러블리 | 반짝반짝 | 대환장 | 마지막 엔딩 톤 차이 |
| D | 초 모양 | 별초 | 하트초 | 숫자초 | 시각적 포인트가 큼 |

권장 초기 적용 세트:

- 1순위: `A 케이크 베이스`
- 2순위: `C 파티 테마`

이유:

- 처음 보는 시청자도 바로 이해할 수 있음
- 결과 비주얼이 분명하게 달라짐
- 스트리머 취향 개입이 쉬움

### Phase 2 후보 테이블

| 세트 | 선택 주제 | 1번 | 2번 | 3번 | 리액션 포인트 |
|---|---|---|---|---|---|
| A | 장식 아이템 | 리본 | 촛불 토퍼 | 폭죽 토퍼 | 안정 vs 축제 vs 난장판 |
| B | 선물 상자 | 작고 귀여운 상자 | 큰 리본 상자 | 정체불명 거대 상자 | 기대감과 개그 |
| C | 파티 소품 | 풍선 | 인형 | 고양이 장식 | 채널 무드와 결합 가능 |
| D | 배경 연출 | 실내 파티룸 | 별빛 무대 | 야외 축제장 | 화면 분위기 전환 |

권장 초기 적용 세트:

- 1순위: `A 장식 아이템`
- 2순위: `C 파티 소품`

## 4. 선택 라운드 운영 규칙

### 유효 투표 기준

- 1라운드 유효 투표자 3명 이상이면 정상 판정
- 3명 미만이면 스트리머가 수동 확정 가능

### 동률 처리

- 1위 동률 발생 시 스트리머 최종 결정
- 이때 화면에는 `스트리머 판정` 배너 표시

### 오늘의 픽 보너스

- 스트리머가 미리 지정한 `오늘의 픽`이 실제 당선되면 준비도 +10
- 당선되지 않아도 결과 화면에서 코멘트 포인트로 사용

## 5. 사고 대응 이벤트 테이블

사고 대응은 초고속 판정보다 `짧은 3회 위기 선택` 구조가 적합하다.

권장 구성:

- 총 3회 사건
- 사건당 5초 ~ 6초
- 각 사건마다 `1`, `2`, `3` 중 하나에 표가 몰리면 방어 성공

### 사고 이벤트 후보

| 이벤트 | 설명 | 채팅 입력 의미 | 성공 효과 | 실패 효과 | 리액션 톤 |
|---|---|---|---|---|---|
| 돌풍 | 촛불이 꺼질 위기 | 좌/중/우 방어 | 안정도 +8 | 안정도 -8 | 긴장감 |
| 고양이 습격 | 케이크를 건드리려 함 | 좌/중/우 차단 | 안정도 +6 | 안정도 -10 | 웃김 |
| 풍선 폭주 | 장식이 날아감 | 좌/중/우 회수 | 안정도 +5 | 안정도 -7 | 소란스러움 |
| 선물 낙하 | 선물상자가 굴러감 | 좌/중/우 받기 | 안정도 +7 | 안정도 -6 | 귀여운 사고 |
| 폭죽 오작동 | 장식 폭죽이 먼저 터짐 | 좌/중/우 정지 | 안정도 +6 | 안정도 -9 | 대환장 |

권장 초기 3종:

- 돌풍
- 고양이 습격
- 선물 낙하

이 세 가지가 가장 읽기 쉽고 리액션도 잘 나온다.

## 6. 사고 대응 라운드 판정 규칙

저채팅량 기준에서는 완전 실패를 남발하면 텐션이 빨리 죽는다.

그래서 아래처럼 3단계 판정이 좋다.

### 성공

- 가장 많은 표가 정답 위치에 몰림
- 안정도 보너스 지급
- 방어 연출 재생

### 반성공

- 표가 적지만 정답 위치가 포함됨
- 안정도 변화 없음 또는 소폭 증가
- 간신히 막은 연출

### 실패

- 정답 위치 득표가 거의 없음
- 안정도 감소
- 사고 연출 재생

권장 최소 판정 기준:

- 사건당 총 유효 입력 3개 이상이면 정상 판정
- 2개 이하면 `반성공 또는 스트리머 스킬 개입 유도`

## 7. 축하 피날레 기준값

현재 운영 규모에서는 총 채팅 수보다 `몇 명이 같이 눌렀는가`가 더 중요하다.

### 권장 성공 조건

| 등급 | 고유 참여자 수 | 총 입력 수 | 결과 |
|---|---:|---:|---|
| 최고 | 12명 이상 | 18개 이상 | 축하열기 대폭 상승 |
| 좋음 | 8명 이상 | 12개 이상 | 축하열기 안정 상승 |
| 보통 | 5명 이상 | 8개 이상 | 최소 성공 |
| 부족 | 4명 이하 | 7개 이하 | 아쉬운 피날레 |

### 운영 원칙

- 최고 등급은 욕심내도 되지만, 최소 성공선은 낮게 잡아야 한다
- 시청자 수 40명 규모에서는 8명 전후 참여만 나와도 성공으로 읽어 주는 편이 맞다

## 8. 결과 등급 및 엔딩 멘트 테이블

낮은 채팅량 환경에서는 결과 멘트가 더 중요하다.  
숫자보다 "이번 판이 어떤 분위기였는지"를 바로 읽게 해야 한다.

| 등급 | 상태 설명 | 결과 멘트 톤 | 방송 리액션 방향 |
|---|---|---|---|
| S | 완벽한 생일 파티 | 모두가 힘을 모아 최고의 생일을 완성함 | 감탄, 감사 |
| A | 아주 좋은 파티 | 약간 정신없었지만 최고의 축하 성공 | 만족, 웃음 |
| B | 성공한 파티 | 우여곡절은 있었지만 생일은 지켜 냄 | 안도, 장난 |
| C | 정신없는 파티 | 사고는 많았지만 어찌저찌 축하는 함 | 폭소, 놀림 |
| D | 대환장 파티 | 파티는 터졌지만 기억에는 남음 | 밈화, 클립화 |

### 결과 멘트 샘플

- S: `오늘은 진짜 사랑받는 생일 파티였어요`
- A: `조금 시끄러웠지만 아주 좋은 파티였어요`
- B: `중간에 흔들렸지만 끝은 해피엔딩이었어요`
- C: `너무 난리였지만 그게 또 우리답네요`
- D: `생일 파티는 망했지만 방송은 성공했어요`

## 9. 스트리머 액션 버튼 매핑

스트리머는 버튼이 많아지면 오히려 진행이 꼬인다.  
초기 MVP는 화면 우측 고정 액션 패널 정도가 적절하다.

| 버튼명 | 사용 구간 | 효과 | 제한 | UI 중요도 |
|---|---|---|---|---|
| 시작 | 오프닝 | 게임 시작 | 판당 1회 | 최고 |
| 오늘의 픽 | 준비 선택 | 특정 선택지 보너스 부여 | 라운드당 1회 | 높음 |
| 리롤 | 준비 선택 | 선택지 세트 재추첨 | 판당 1회 | 높음 |
| 생일 소원 | 사고 대응 | 사건 1회 성공 보정 | 판당 1회 | 최고 |
| 피날레 발동 | 축하 피날레 | 최종 연출 발동 | 조건 충족 시 1회 | 최고 |
| 결과 코멘트 | 결과 공개 | 베스트 순간 문구 선택 | 판당 1회 | 중간 |

## 10. 스트리머 액션 UI 배치안

### 우측 패널 고정안

- 상단: 현재 라운드
- 중단: 사용 가능한 스트리머 버튼
- 하단: 남은 개입 횟수

이 배치가 좋은 이유는 아래와 같다.

- 메인 화면을 가리지 않음
- 방송 중 마우스로 누르기 쉬움
- 어떤 버튼이 현재 활성화됐는지 직관적임

### 버튼 상태 규칙

- 비활성: 회색
- 사용 가능: 채널 컬러 강조
- 긴급 사용 추천: 반짝임 또는 점멸
- 사용 완료: 체크 또는 소모 표시

## 11. 저채팅량 대응 Fallback 규칙

이 문서에서 가장 중요한 부분이다.

### 선택 라운드 fallback

- 총 투표 0개: 스트리머 자동 선택
- 총 투표 1~2개: 스트리머 판정 + `채팅이 적어서 직접 골랐어요` 연출
- 동률: 스트리머 판정

### 사고 대응 fallback

- 유효 입력 0개: 반실패
- 유효 입력 1~2개: 스트리머 스킬 개입 유도
- 연속 무응답 2회: 다음 사고는 자동 난이도 완화

### 피날레 fallback

- 참여자 수 3명 이하: 실패로 닫지 말고 `소박한 축하 엔딩`
- 스트리머가 `피날레 발동`을 눌러도 게이지 부족이면 소규모 연출만 재생

이 구조가 있어야 사람이 적은 날에도 방송이 민망해지지 않는다.

## 12. 권장 초기 콘텐츠 조합

MVP 첫 버전은 아래 조합이 가장 안전하다.

### 준비 선택

- Phase 1: 케이크 베이스
- Phase 2: 장식 아이템

### 사고 이벤트

- 돌풍
- 고양이 습격
- 선물 낙하

### 스트리머 액션

- 시작
- 오늘의 픽
- 리롤
- 생일 소원
- 피날레 발동

이 조합은 복잡하지 않고, 저채팅량에서도 반응 포인트가 잘 산다.
