# 전통 2D 쯔꾸르용 이미지 프롬프트 팩

- Version: 3.0
- Date: 2026-04-16
- Project id: `zunyang-party-panic`
- Purpose: ChatGPT 이미지 생성으로 `전통적인 2D 쯔꾸르 스타일` 팬게임에 필요한 자산을 만들 수 있도록 프롬프트를 정리한다.

## 1. 먼저 정리

이 문서는 `와이드 일러스트 배경 중심` 문서가 아니다.

이 문서가 목표로 하는 건 아래다.

- 전통적인 2D 쯔꾸르풍
- 타일 기반 맵 구성
- 4방향 플레이어 스프라이트
- 개별 오브젝트 스프라이트
- 대화창용 페이스
- 필요할 때만 이벤트 CG

즉, `배경 한 장`보다 `맵 조립용 자산`이 중심이다.

## 2. 전통적인 2D 쯔꾸르 기준

이번 프로젝트에서 말하는 전통적인 2D 쯔꾸르는 아래로 통일한다.

- 시점: 위에서 내려다보는 오버헤드 2D, 약한 3/4 느낌
- 맵 제작 방식: 타일 기반
- 캐릭터 표현: SD 또는 치비 스프라이트
- 대화 표현: 페이스 포트레이트 + 대사창
- 이벤트 표현: 오브젝트 조사, 문 이동, 짧은 컷신
- 색감: 따뜻한 생일 방송 분위기

## 3. ChatGPT 이미지 생성 사용 방식

1. 새 이미지 생성 대화를 연다.
2. `asset` 폴더에 넣어 둔 치즈냥 캐릭터 레퍼런스를 전부 업로드한다.
3. 아래 `세션 시작 프롬프트`를 먼저 넣는다.
4. 자산을 `하나씩` 생성한다.
5. 같은 세션에서 계속 생성한다.

중요:

- 캐릭터 일관성을 위해 반드시 같은 세션에서 이어서 만든다.
- 한 번에 여러 자산을 요구하지 말고, `하나씩` 생성한다.
- 스프라이트 시트가 불안하면 먼저 `방향별 단일 포즈`를 만들고 나중에 시트로 묶는다.

## 4. 아트 방향

### 캐릭터 고정 요소

- 여성 한국 VTuber 느낌
- 아주 밝은 금발 또는 크림 블론드 헤어
- 큰 동물 귀
- 빨간 리본이 들어간 옆머리 포인트
- 보라색 머리핀과 별 장식
- 붉은기 있는 핑크 레드 눈
- 귀엽고 부드러운 인상

### 세계관 고정 요소

- 생일 방송을 앞둔 스트리머 공간
- 핑크, 크림, 옅은 금색, 코랄, 연한 민트
- 따뜻한 실내 조명
- 과한 사이버, SF, 공포, 메탈릭 금지

### 쯔꾸르용 제약

- 오브젝트는 정면 일러스트보다 `맵 위에 놓을 수 있는 시점`이어야 한다
- 타일은 perspective가 강하면 안 된다
- 오브젝트는 배치 가능한 크기와 실루엣이 중요하다
- 화면 전체 일러스트보다 `분리 가능한 자산`이 우선이다

## 5. 세션 시작 프롬프트

아래 프롬프트를 이미지 생성 대화 첫 메시지로 넣는다.

```text
I am creating assets for a traditional 2D RPG Maker style Korean VTuber birthday fan game.
Use the uploaded reference images as the exact character identity anchor for all outputs in this chat.

Main character identity:
- female Korean VTuber
- very light blonde or cream blonde hair
- large soft animal ears
- red ribbon detail near a side braid area
- purple hairpins and small star accessory
- warm pink-red eyes
- cute gentle polished anime face

Global game direction:
- traditional 2D top-down RPG Maker style
- tile-based map construction
- chibi or SD readable sprites
- warm birthday livestream preparation mood
- cute but slightly emotional
- pastel pink, cream, pale gold, soft coral, light mint

Important rules:
- preserve the same character identity in the entire session
- prefer modular game-ready assets over cinematic full-scene art
- use clean top-down or RPG-style object angles
- avoid dramatic perspective
- avoid photorealism
- no text
- no watermark
- no logo
- one asset only per prompt
```

## 6. 제작 우선순위

이 순서가 가장 안전하다.

1. 페이스 포트레이트 기준 고정
2. 플레이어 SD 스프라이트 기준 고정
3. 허브 방 타일셋
4. 오브젝트 세트
5. 챕터별 맵 타일셋
6. UI 윈도우 스킨
7. 이벤트 CG

## 7. 출력 규격 추천

### 페이스 포트레이트

- 권장: `1024x1024`
- 배경 단순
- bust shot 또는 shoulders-up

### 플레이어 스프라이트

- 권장: 방향별 개별 PNG
- 나중에 직접 시트로 묶기 쉬운 단색 배경
- 작은 크기로 축소해도 읽히게

### 타일셋

- 목표 기준: `48x48 tile compatibility`
- 결과물이 정확한 시트로 안 나오면, `반복 가능한 타일 느낌의 조각 세트`로 먼저 만든다

### 오브젝트 세트

- 각 오브젝트가 분리된 상태가 중요
- top-down RPG map angle 유지

### 이벤트 CG

- 비율: `16:9`
- 컷신용으로만 사용
- 너무 많이 만들 필요 없음

## 9. 수정 프롬프트 템플릿

### 쯔꾸르 시점 보정

```text
Make the asset more suitable for a traditional 2D RPG Maker style game. Reduce cinematic perspective and make the object angle cleaner and more top-down or map-friendly.
```

### 캐릭터 일관성 보정

```text
Keep exactly the same face, hair color, ears, ribbon detail, and accessory placement from the uploaded references and previous successful outputs in this chat. Do not redesign the character.
```

### 오브젝트 분리도 보정

```text
Separate the objects more clearly so they feel easier to use as modular game assets. Make silhouettes cleaner and spacing clearer.
```

### 타일셋 느낌 보정

```text
Make the image feel less like one illustration and more like a modular tileset or map asset resource for a traditional 2D RPG Maker style game.
```

## 10. 실전 메모

- 진짜 전통 쯔꾸르로 가려면 `배경 한 장`보다 `맵 조립용 자산`이 우선이다.
- ChatGPT가 정확한 타일 시트를 완벽히 맞추지 못할 수 있으니, 먼저 `오브젝트 분리도 높은 자산`을 만들고 나중에 직접 정렬하는 쪽이 안정적이다.
- 가장 먼저 고정해야 하는 건 `페이스 포트레이트`와 `SD 플레이어 스프라이트`다.
- 그 다음부터는 맵과 오브젝트를 전부 그 기준에 맞춰 붙이면 된다.
