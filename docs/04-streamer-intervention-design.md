# Streamer Intervention Design

- Version: 0.1
- Date: 2026-04-16
- Based on: `docs/01-birthday-chat-game-plan.md`, `docs/02-birthday-game-genre-direction.md`, `docs/03-birthday-game-loop-design.md`
- Goal of this document: 스트리머가 단순 진행자가 아니라 시스템적으로 게임에 개입하는 구조를 정의한다.

## 1. 문제 정의

현재까지의 기획은 채팅 상호작용 구조는 잘 잡혀 있지만, 스트리머는 멘트를 얹는 역할에 가깝다.

하지만 이 프로젝트는 일반 투표 게임이 아니라 `스트리머 생일기념 팬게임`이다.  
따라서 스트리머는 단순한 관전자나 사회자가 아니라, 팬들과 함께 판을 굴리는 핵심 플레이어여야 한다.

즉, 이 게임은 아래 구조로 정리하는 것이 맞다.

- 채팅: 집단 의사결정과 에너지 공급 담당
- 스트리머: 판의 방향을 흔들고 완성시키는 연출자 겸 플레이어

## 2. 스트리머 역할 정의

스트리머의 역할은 아래 4가지로 나눈다.

### 1. 진행자

판을 시작하고, 라운드 흐름을 끊지 않게 이어 가는 역할이다.

### 2. 판정자

채팅이 만들어 낸 결과에 의미를 붙이고, 방송적으로 맛을 살리는 역할이다.

### 3. 개입자

위기 상황이나 선택 구간에서 제한적으로 시스템에 손을 대는 역할이다.

### 4. 피날레 트리거

마지막 축하 장면을 직접 완성시키는 역할이다.

이 네 역할이 있어야 스트리머가 "옆에서 보는 사람"이 아니라 "판 중심에 있는 사람"이 된다.

## 3. 개입 설계 원칙

스트리머 개입은 강해야 하지만, 채팅 주도권을 뺏으면 안 된다.

그래서 아래 원칙으로 잡는 것이 적절하다.

### 원칙 1. 스트리머는 흐름을 바꿀 수 있지만, 다수결을 자주 뒤엎지 않는다

채팅이 고른 결과를 매번 뒤집으면 시청자 개입감이 무너진다.  
스트리머는 방향을 기울이거나, 보너스를 주거나, 위기를 구하는 식으로 개입해야 한다.

### 원칙 2. 스트리머 개입은 눈에 띄게 보여야 한다

스트리머가 버튼을 눌렀다면 화면도 바로 반응해야 한다.

- 배너
- 색 변화
- 효과음
- 캐릭터 리액션

이런 즉시 피드백이 필요하다.

### 원칙 3. 스트리머 개입은 제한되어야 한다

무한 개입이 가능하면 게임이 채팅 게임이 아니라 스트리머 솔로 게임이 된다.  
따라서 각 기능은 횟수 또는 조건 제한이 필요하다.

### 원칙 4. 개입 자체가 방송 포인트가 되어야 한다

버튼이 단순 편의기능이면 안 된다.  
스트리머가 "지금 쓸까 말까" 고민하고, 채팅이 "지금 쓰세요"라고 반응할 수 있어야 한다.

## 4. 추천 개입 구조

현재 기준 추천은 `판당 5종 개입` 구조다.

1. 시작 선언
2. 오늘의 픽
3. 리롤
4. 위기 대응 스킬
5. 피날레 트리거

이 다섯 개면 과하지 않으면서도 충분히 존재감이 생긴다.

## 5. 개입 포인트 상세

### A. 시작 선언

스트리머가 직접 판을 시작한다.

- 기능: `시작` 버튼
- 효과: 카운트다운 시작
- 의미: 게임의 리듬을 스트리머가 쥔다

이건 단순한 버튼 같지만, 방송 템포를 스트리머가 직접 잡는다는 점에서 중요하다.

### B. 오늘의 픽

준비 선택 라운드에서 스트리머가 한 선택지에 `오늘의 픽` 표시를 붙인다.

- 기능: 세 선택지 중 하나 강조
- 효과:
  - 채팅이 그 선택지를 고르면 준비도 추가 보너스
  - 실패해도 결과 멘트에서 리액션 포인트 생성
- 제한: 선택 라운드마다 1회

이 기능의 장점은 `채팅 다수결`을 유지하면서도 스트리머의 취향과 캐릭터를 게임 안에 녹일 수 있다는 점이다.

### C. 리롤

준비 라운드에서 보기 구성이 마음에 들지 않을 때 한 번 다시 뽑는다.

- 기능: `리롤` 버튼
- 효과: 현재 선택지 3개를 새 세트로 교체
- 제한: 판당 1회

이 기능은 스트리머가 판을 좀 더 자기 방송다운 방향으로 잡게 해 준다.

### D. 위기 대응 스킬

사고 대응 라운드에서 스트리머가 직접 쓰는 능동 개입이다.

추천 스킬 구조는 아래와 같다.

- `생일 소원`
  - 효과: 이번 판정 1회 자동 보정
  - 제한: 판당 1회
- `촛불 보호`
  - 효과: 특정 위치 3초 보호
  - 제한: 판당 1회
- `장난 금지`
  - 효과: 사고 속도 잠시 완화
  - 제한: 판당 1회

실제 구현에서는 이 중 1~2개만 먼저 넣어도 충분하다.

### E. 피날레 트리거

채팅이 축하 게이지를 채우면, 마지막 폭발 연출은 스트리머가 직접 터뜨린다.

- 기능: `피날레 발동` 버튼
- 조건: 축하 게이지 임계치 도달
- 효과: 폭죽, 컷신, 환호 연출, 최종 보너스

이 기능이 중요한 이유는 마지막 순간을 스트리머 손으로 완성시키기 때문이다.

## 6. 권장 개입 비율

채팅과 스트리머의 영향력 비율은 아래 정도가 적절하다.

- 채팅 영향력: 70
- 스트리머 영향력: 30

이 비율을 유지해야 아래 두 가지가 동시에 성립한다.

- 시청자는 "우리가 만들었다"고 느낀다
- 스트리머는 "내가 직접 굴렸다"고 느낀다

## 7. 스트리머 개입이 만들어 내는 방송 장면

이 시스템이 잘 작동하면 아래 장면들이 나온다.

- 스트리머가 싫어하는 선택지를 채팅이 일부러 고름
- 스트리머가 오늘의 픽을 밀었는데 채팅이 배신함
- 위기 상황에서 스트리머가 스킬을 아껴야 할지 고민함
- 채팅이 "지금 스킬 쓰세요"라고 몰아감
- 마지막 피날레를 스트리머가 직접 터뜨리며 엔딩 완성

즉, 개입 버튼은 시스템 기능이면서 동시에 방송 대사와 클립 포인트를 만드는 장치다.

## 8. 피해야 할 구조

아래 방향은 피하는 것이 좋다.

### 1. 스트리머가 채팅 다수결을 매번 뒤집는 구조

이 경우 시청자 참여감이 크게 떨어진다.

### 2. 스트리머 개입이 너무 약해서 멘트만 남는 구조

이 경우 일반 투표형 오버레이와 차별성이 약하다.

### 3. 스트리머가 너무 바빠서 방송 진행이 꼬이는 구조

버튼 수가 너무 많으면 오히려 리액션이 죽는다.

## 9. 초기 MVP 기준 추천안

초기 버전에서는 아래 정도만 넣는 것이 적절하다.

- 시작 버튼
- 오늘의 픽
- 리롤 1회
- 위기 대응 스킬 1종
- 피날레 발동 버튼

즉, `간단하지만 존재감 있는 개입`이 중요하다.

## 10. 이번 단계 결론

현재 기준 스트리머 개입 구조는 아래 문장으로 정리할 수 있다.

`이 게임에서 스트리머는 사회자가 아니라, 채팅이 만든 흐름을 흔들고 완성하는 제한적 개입 권한을 가진 플레이어이자 연출자다.`
